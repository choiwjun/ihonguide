---

## 1. 문제 정의

### 1.1 현재 상황

이혼을 고민하거나 준비하는 사람들은 다음과 같은 어려움을 겪고 있습니다:

| 문제 | 현재 상태 | 영향 |
|------|----------|------|
| 정보 파편화 | 블로그, 지식인, 카페에 흩어진 정보 | 신뢰할 수 있는 정보 찾기 어려움 |
| 막막함 | 뭐부터 해야 할지 모름 | 결정 지연, 불안 증가 |
| 높은 문턱 | 변호사 상담은 심리적 부담 | 전문가 도움 받기 주저 |
| 비용 불안 | 얼마나 들지 예측 불가 | 재정적 두려움 |

### 1.2 핵심 문제 (Primary Problem)

> **"뭐부터 해야 할지 모르는 막막함"**
>
> 이혼을 결심하거나 고민 중인 사람이 첫 발을 떼지 못하고 있다.

### 1.3 해결 가설

> 체계적인 정보와 실용적인 도구를 무료로 제공하면, 사용자는 막막함을 해소하고 차분하게 이혼 준비를 시작할 수 있을 것이다. 이 과정에서 신뢰가 형성되면, 자연스럽게 전문가 상담으로 연결될 것이다.

## 2. 제품 비전

### 2.1 한 줄 정의

**"이혼을 고민하거나 준비하는 사람들에게 체계적인 정보와 도구를 무료로 제공하는 플랫폼"**

### 2.2 핵심 가치 제안

| 기존 경험 | 이혼준비의 해결책 |
|----------|------------------|
| "뭐부터 해야 하지?" | → 이혼 유형 진단으로 첫 방향 제시 |
| "양육비는 얼마나 받을 수 있지?" | → 양육비 계산기로 즉시 예상 |
| "변호사한테 바로 연락하기 부담돼" | → 충분히 정보 습득 후 편하게 상담 신청 |

### 2.3 궁극적 목표

> 막막했던 이혼 준비 과정을 차분하게 정리하고 첫 발을 뗄 수 있게 돕는다.

## 3. 사용자 페르소나

### 3.1 Primary Persona

| 항목 | 내용 |
|------|------|
| **이름** | "준비하는 사람" |
| **핵심 특성** | 이혼을 계획하고 준비 중이지만, 바로 변호사와 상담하기는 부담되는 사람 |
| **연령** | 연령 무관 (30~50대 중심) |
| **공통 니즈** | 절차와 비용을 먼저 파악하고 싶음 |
| **심리 상태** | 불안, 막막함, 정보에 대한 갈증 |
| **행동 패턴** | 검색으로 정보 수집 → 도구로 내 상황 파악 → 준비 후 전문가 연결 |

### 3.2 사용 맥락

- **기기:** 스마트폰과 PC 반반
- **시간:** 특정 시간대 없음, 생각날 때 수시로 접속
- **환경:** 혼자 있을 때, 조용히 정보 탐색

### 3.3 사용자 여정 단계

```
[인지] "이혼 절차" 검색 → 콘텐츠 유입
         ↓
[관심] "내 상황은?" → 이혼 유형 진단
         ↓
[탐색] "더 자세히" → 양육비 계산, 가이드 읽기
         ↓
[신뢰] "여기 정보 괜찮네" → 재방문, 추가 탐색
         ↓
[전환] "전문가한테 물어보고 싶다" → 상담 신청
```

## 4. 사용자 스토리

### FEAT-1: 이혼 유형 진단

| ID | 사용자 스토리 | 인수 조건 |
|----|--------------|----------|
| US-1.1 | 사용자로서, 나는 간단한 질문에 답하면 내 상황에 맞는 이혼 유형(협의/조정/소송)을 알고 싶다 | 10문항 이내, 3분 내 완료 가능 |
| US-1.2 | 사용자로서, 나는 진단 결과와 함께 다음 단계 안내를 받고 싶다 | 결과 페이지에 권장 행동 표시 |
| US-1.3 | 사용자로서, 나는 결과를 저장하거나 다시 볼 수 있으면 좋겠다 | 로그인 시 결과 저장 |

### FEAT-2: 양육비 계산기

| ID | 사용자 스토리 | 인수 조건 |
|----|--------------|----------|
| US-2.1 | 사용자로서, 나는 내 소득과 자녀 정보를 입력하면 예상 양육비를 알고 싶다 | 법원 양육비 산정기준표 기반 계산 |
| US-2.2 | 사용자로서, 나는 계산 근거를 이해할 수 있도록 설명을 보고 싶다 | 산정 기준 및 참고 사항 표시 |
| US-2.3 | 사용자로서, 나는 여러 시나리오를 비교해보고 싶다 | 조건 변경 후 재계산 가능 |

### FEAT-3: 상담 신청 폼

| ID | 사용자 스토리 | 인수 조건 |
|----|--------------|----------|
| US-3.1 | 사용자로서, 나는 연락처만 남기면 변호사가 연락해주길 원한다 | 필수: 이름, 연락처, 희망 시간대 |
| US-3.2 | 사용자로서, 나는 내 상황을 미리 전달해서 효율적인 상담을 받고 싶다 | 선택: 상황, 관심사, 간단 설명 |
| US-3.3 | 사용자로서, 나는 상담 신청 후 언제 연락이 올지 알고 싶다 | 신청 완료 페이지에 예상 연락 시간 안내 |

## 7. 가정과 리스크

### 7.1 핵심 가정

| 가정 | 검증 방법 |
|------|----------|
| 사용자는 무료 도구를 신뢰하고 사용할 것이다 | 진단/계산기 사용률 측정 |
| 도구 사용 후 상담 신청으로 전환될 것이다 | 전환율 측정 |
| 파트너 변호사가 리드에 만족할 것이다 | 파트너 피드백 |

### 7.2 리스크 매트릭스

| 리스크 | 가능성 | 영향 | 완화 방안 |
|--------|-------|------|----------|
| 변호사법 위반 우려 | 낮음 | 높음 | 결제 기능 없음, 정보 제공만, 면책 고지 |
| SEO 효과 부진 | 중간 | 중간 | 양질의 콘텐츠, 다채널 마케팅 |
| 리드 품질 불만 | 중간 | 중간 | 상담 폼 최적화, 필터링 |
| 개인정보 유출 | 낮음 | 높음 | 암호화, 최소 수집, Supabase RLS |
